Semantic Elements

In simple words semantic elements are elements which a browser can read. Semantic elements are used to define a specific part of a browser.Screen reader uses semantic elements as a signpost to help visually impaired users to navigate a web page. Using semantic elements can help influence the page's search ranking in a positive way.It doesnot confuses other devs with endless div's. It is a best practice to use semantic elements while developing a web page.
Here are some semantic elements:

<nav>

<header>

<section>

<article>

<aside>

<footer>


Inline Elements

Elements which take only a specific part of a browser and not the whole block are called as inline elements. <span> is a example of inline element.Lets demonstrate it with an example:

<div>This is an example of <span>inline</span> element.

Output

This is an example of inline element.

As we can see from the above output the span tag only acquires the highlighted width of the browser.Therefore they are called as inline element

Here are some inline elements:

<a>

<img>

<button>

<span>

<small>

Block Elements

A block element always starts on a new line.It takes full width which is availble.<p> is a block level element. Lets demonstrate it with an example:

<div>This is an example of <p> block </p> element</div>

Output

This is an example of

block

element.

In the above example the <p> tag acquires a whole highlighted block of the browser.

Here are some block elements:

<li>

<ol>

<p>

<h1>

<div>
